# 📌 DoR e DoD da Sprint 2

### **MVP da Sprint 2:**  
2️⃣ Sprint II: Inteligência analítica e alertas > Ranking de oportunidades por região com indicadores combinados.


### User Stories da Sprint 2

4. **Como analista de crédito da DM, quero visualizar um ranking de oportunidades por região, baseado nos indicadores de crédito, inadimplência e econômicos, para orientar decisões de expansão.**  
   - **DoR:** Regra de cálculo do ranking definida (peso dos indicadores); endpoint da API planejado para ranking; estrutura do front preparada para exibição de ranking. 
   - **DoD:**  
     - [x] Regra de cálculo do ranking implementada no backend.
     - [x] Endpoint da API retornando ranking.
     - [x] Ranking exibido no front (lista ou gráfico).  



5. **Como analista de crédito da DM, quero visualizar dashboards interativos com indicadores combinados (concessão x inadimplência x potencial), para identificar oportunidades sustentáveis de expansão.**  
   - **DoR:** Indicadores já integrados e acessíveis via API; definição de como os indicadores serão combinados; layout do dashboard aprovado; biblioteca de gráficos definida; endpoints preparados para múltiplos indicadores.  
   - **DoD:**  
     - [x] API com endpoint retornando indicadores combinados.
     - [x] Dashboard com múltiplos gráficos (ex: comparação entre indicadores) implementado.
     - [x]Integração front-end + API funcionando sem erros.



6. **Como analista de crédito da DM, quero receber alertas visuais para indicadores críticos (ex: inadimplência acima da média), para rápida identificação de riscos.**  
   - **DoR:** Regras de definição de “indicador crítico” estabelecidas; valores médios ou limites já calculáveis; componentes visuais de alerta definidos (cores, ícones, etc.); dados disponíveis no dashboard.
   - **DoD:**  
     - [x] Lógica de identificação de indicadores críticos implementada.
     - [x] Alertas visuais exibidos no front (cores, badges ou ícones).
     - [x] Casos de teste validados (ex: acima/abaixo da média).
